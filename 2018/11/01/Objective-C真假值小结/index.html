在Objective-C中，BOOL其实是宏定义，在objc.h头文件的源代码中它被定义为signed char类型，见代码77-84行。（注：在64位的iPhone平台以及iWatch平台下，BOOL等价于bool，见代码77-79行）

在Objective-C中，广泛使用的YES和NO也只是signed char类型的1和0，见代码88-94行。

撸码容易遇到的坑

上述，Objective-C中的 BOOL 是 signed char类型，因此 BOOL 类型的值范围是 -128-127 不只是 1(YES)/0(NO)，其中0表示值为假，其他都为真。换句话说，BOOL 类型的真值不一定是 1。所以应当注意

  • 避免与真值YES直接比较
  • bool 类型值可以与true直接进行比较
  • BOOL bool Boolean 类型间不要随便转换
